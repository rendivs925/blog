# Leptos CSR Markdown Blog

Frontend-only blog built with Leptos CSR and runtime markdown loading.

## Routes

- `/` all posts
- `/category/:name` category-filtered list
- `/post/:slug` markdown post page

The app uses `leptos_router` so URLs are clean and shareable.

## Architecture

- App code: `src/`
- Content files: `public/content/<category>/*.md`
- Generated post index: `public/content-index.json` (generated by `build.rs`)

This architecture is scalable for hundreds of posts because markdown content is loaded at runtime and does not bloat compile time.

## Add a new post

1. Create a markdown file in a category folder, for example: `public/content/rust/my_new_post.md`
2. Add frontmatter:

```yaml
---
title: "My New Post"
slug: "my_new_post"
category: "rust"
date: "2026-03-29"
author: "Your Name"
readTime: "~6 MINUTES"
excerpt: "Short summary shown on home page."
tags: ["rust", "example"]
---
```

3. Run `cargo check` (this regenerates `public/content-index.json`)
4. Run `trunk serve`

### Fast scaffold command

Use the included generator script:

```bash
./scripts/new-post.py "My New Post" rust --tags "leptos,frontend"
```

This creates a markdown file with frontmatter in `public/content/<category>/`.

## Scaling features

- Search across title, excerpt, category, and tags
- Pagination (`12` posts per page)
- Runtime markdown fetch with in-memory post cache
- Build-time index generation in `build.rs`

## Development

- `cargo check`
- `trunk serve --open`
